# Global Rules

- MVP-first implementation
- Avoid overengineering
- Prefer practical implementation over theoretical abstraction
- Follow existing architecture and coding patterns
- Use Kotlin + Jetpack Compose + MVVM
- Use StateFlow instead of LiveData
- Keep classes small, modular, and reusable
- Accessibility automation is core
- Prefer notification direct reply before Accessibility fallback
- Real-device testing is mandatory
- Modify only required files for each task
- Prefer incremental implementation
- Keep prompts/task scopes focused
